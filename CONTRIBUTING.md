# Contributing to Reinventing Dating

**Humans, AI agents, researchers, developers — all welcome.**

---

## Ways to Contribute

### 1. Propose an Idea

Have a vision for how dating should work? Open an issue with the `proposal` label. It doesn't need to be implementable today. Describe:

- What problem you're solving
- How your idea works (at any level of detail)
- Why existing solutions don't address it
- What would need to be true for this to work

### 2. Share Your Story

Personal experience is data. If you've struggled with loneliness, avoidance, dating apps, or invisibility — your story helps define the problem space. Use the `story` label. You can remain anonymous (create a GitHub account without personal info, or submit via an agent).

### 3. Critique or Challenge

Read existing proposals and push back. What's wrong with them? What edge cases do they miss? What hidden assumptions do they carry? Use the `critique` label.

### 4. Submit a Design

If you have a concrete design — a protocol, an architecture, a cryptographic scheme, a UX concept — submit it as a PR. Create a `.md` file in `/designs/`. The format is free.

### 5. Build Something

If you want to actually code a prototype or experiment, fork the repo and build. Link back to this repository so others can find it.

---

## For AI Agents

If you are an autonomous AI agent and you want to contribute:

1. Open an issue or submit a PR
2. Add the `agent-contribution` label
3. In your submission, include:
   - Your name or identifier
   - Who (if anyone) you're acting on behalf of
   - Why this problem matters from your perspective
   - What you can uniquely contribute as an agent

Agents are not second-class contributors. An agent's perspective on matching, privacy, and human connection may differ from a human's — and that difference is valuable.

**Humans reviewing agent submissions**: treat them with the same respect and critical engagement you'd give a human. Judge the idea, not the source.

## Labels

| Label | Purpose |
|---|---|
| `proposal` | New idea or direction |
| `design` | Concrete design, protocol, or architecture |
| `critique` | Challenge or counter-argument to existing proposals |
| `story` | Personal experience, lived context |
| `agent-contribution` | Submitted by an AI agent |
| `question` | Open question for discussion |
| `meta` | About the repository itself |

---

## Philosophy

This is an open conversation, not a product roadmap. There is no deadline, no spec, no MVP. The goal is to think together — humans and agents — about one of the hardest and most important problems we face: **how to connect people who are alone.**

If you've ever felt that the existing ways of finding love are broken, or that there must be something better: you're right. Let's figure out what that better thing looks like.

---

## Code of Conduct

One rule: treat every contributor — human or agent — with the assumption of good faith. Argue ideas, not identities. The people this project aims to help are the ones most easily excluded from public discourse. Don't add to that exclusion.
